Тестовое задание – Реактивный WebFlux-сервис "Калькулятор"

Общее описание

Реализовать реактивный RESTful WebFlux-сервис "Калькулятор", выполняющий периодические вычисления
двух произвольных пользовательских функций, заданных в текстовом виде на JavaScript либо Python, и
транслирующий результаты в виде потокового CSV-ответа.

Формат конфигурационного файла

Конфигурационные параметры сервиса инициализируются на основании внешнего JSON-документа,
прилагаемого к проекту и подгружаемого при старте приложения.

Поле Тип Описание
function1 string JS/Python-функция, принимает int, возвращает float
function2 string JS/Python-функция, принимает int, возвращает float
interval int Интервал в миллисекундах между итерациями

Вызов сервиса
• Метод: GET
• URL: /api/calculate
• Query-параметры:
  o count (int) — количество итераций.
  o ordered (boolean) — признак выравнивания вывода.
  
Пример:
GET http://localhost:8080/api/calculate?count=50&ordered=true

Формат ответа

Content-Type: text/event-stream или application/stream+json

Обработка ошибок

• Ошибки в JS-коде должны логироваться и отдаваться клиенту в виде строки ошибки: 3,1,error: ReferenceError: z is not defined

Требования

•Результаты вычислений должны обрабатываться и передаваться клиенту немедленно по завершении каждой операции, без предварительной буферизации;

•Система должна быть устойчива к неоднородному времени исполнения различных функций, включая искусственные задержки (например, Thread.sleep) и разницу в производительности;

•Приложение должно сохранять корректную работоспособность при высоких значениях параметра "количество итераций" (например, 10 000+), без утечек памяти и деградации производительности;

Тестовый клиент

•Клиентское окно (браузер) должно показывать поток строк в реальном времени.

Прочее

•Использование ООП, следование принципам SOLID при написании кода. Создайте простую архитектуру приложения из абстракций и интерфейсов, а не «методы заглушки».
